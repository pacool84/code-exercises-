## ✨ ¿Que es este proyecto?

Este repo es mi laboratorio personal de practica tecnica.

La idea no es solo acumular ejercicios resueltos de HackerRank, sino construir un registro de aprendizaje que se vea bien, se entienda rapido y ademas sea util para volver a estudiar despues.

Cada problema intenta dejar tres cosas:

- una implementacion funcional
- una guia de estudio en `.md`
- una forma mas clara de pensar el problema

En otras palabras: menos "ya lo resolvi", mas "ya entendi como resolver este tipo de problema".

> Este no es un cementerio de snippets. Es un log vivo de practica con criterio.

## 🔥 ¿Por que llama la atencion este repo?

- Porque no se queda en la respuesta final: muestra proceso.
- Porque mezcla codigo y explicacion, no solo codigo.
- Porque convierte ejercicios clasicos en material de estudio reusable.
- Porque esta construido con constancia, no con relleno.

## ⚡ El mood del proyecto

> Resolver retos esta bien. Entender por que la solucion funciona esta mejor.

Este repositorio busca exactamente eso:

- practicar algoritmos
- mejorar la lectura de problemas
- dominar input/output estilo HackerRank
- escribir soluciones limpias
- documentar el aprendizaje sin hacerlo pesado

## 📝 ¿Que hace especial a las guias `.md`?

No estan ahi para decorar el repo.

Cada guia intenta responder preguntas como:

- ¿que pide realmente el problema?
- ¿que conceptos necesitas antes de resolverlo?
- ¿como se traduce el enunciado a pasos logicos?
- ¿que errores son faciles de cometer?
- ¿que deberias revisar antes de darlo por bueno?

La meta es que cada archivo `.md` sirva como un repaso rapido y tambien como una ayuda real si vuelvo al problema semanas despues.

## 🚀 Ejecutar los ejercicios

### JavaScript

Muchos archivos siguen el formato original de HackerRank y usan `OUTPUT_PATH`, asi que en varios casos estan pensados para ejecutarse directamente en la plataforma o con una adaptacion minima en local.

Referencia general:

```bash
node javascript/09-grading-students.js
```

### Java

Compilacion base:

```bash
javac java/01-simple-array-sum.java
```

Nota: algunos archivos siguen la estructura que HackerRank espera, por lo que en local puede ser necesario ajustar nombres de archivo o clase publica.
